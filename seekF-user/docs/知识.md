# 1.localStorage、sessionStorage、cookie三者消失场景对比表



| 触发操作                | localStorage | sessionStorage | cookie（设过期时间） | cookie（未设过期时间） |
| ----------------------- | ------------ | -------------- | -------------------- | ---------------------- |
| 关闭当前标签页          | ❌ 保留       | ✅ 消失         | ❌ 保留               | ❌ 保留（浏览器未关）   |
| 关闭浏览器              | ❌ 保留       | ✅ 消失         | ❌ 保留               | ✅ 消失                 |
| 刷新页面                | ❌ 保留       | ❌ 保留         | ❌ 保留               | ❌ 保留                 |
| 清除浏览器数据          | ✅ 消失       | ✅ 消失         | ✅ 消失               | ✅ 消失                 |
| 代码主动删除            | ✅ 消失       | ✅ 消失         | ✅ 消失               | ✅ 消失                 |
| 过期时间到（仅 cookie） | ❌ 保留       | ❌ 保留         | ✅ 消失               | ❌ 无此场景             |



# 2.图标在页面刷新后未正确显示的问题

使用 v-if/v-else 切换两个独立的 Icon 组件。这样 Vue 会完全销毁旧组件并创建新组件，而不是尝试更新已有组件的 name  属性（这可能有缓存问题）。  

```
<Icon :name="isLiked ? 'mdi:heart' : 'mdi:heart-outline'" />   
改成
<Icon v-if="isLiked" name="mdi:heart" />                                                                
<Icon v-else name="mdi:heart-outline" />                                                         
```
