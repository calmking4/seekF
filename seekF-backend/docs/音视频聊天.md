## 概述

本项目实现了基于 WebRTC 的一对一音视频通话功能，支持：
- 视频通话（含音频）
- 通话邀请/接受/拒绝
- 通话中静音、关闭摄像头、挂断
- 左右并排显示本地和远程视频

```
│                         后端 (Go)                            │
├─────────────────────────────────────────────────────────────┤
│  server.go             - WebSocket 服务器                   │
│  av_data_request.go    - 信令数据结构                       │
│  Kafka                 - 消息队列（信令中转）               │
└─────────────────────────────────────────────────────────────┘
```

### 核心技术
- **WebRTC**: 实现 P2P 音视频传输
- **WebSocket**: 实时信令交换
- **Kafka**: 消息队列，解耦信令处理
- **STUN/TURN**: NAT 穿透（本项目使用免费 Google STUN）

```go
package userreq

// AVData 音视频通话数据
type AVData struct {
    MessageId string        `json:"message_id"`   // SIGNAL (信令) 或 PROXY (需要存储)
    Type      string        `json:"type"`         // 信令类型
    Sdp       *SdpData      `json:"sdp,omitempty"`
    Candidate *IceCandidate `json:"candidate,omitempty"`
}

// SdpData SDP 会话描述
type SdpData struct {
    Type string `json:"type"` // offer 或 answer
    Sdp  string `json:"sdp"`
}

// IceCandidate ICE 候选地址
type IceCandidate struct {
    Candidate     string `json:"candidate"`
    SdpMLineIndex int    `json:"sdpMLineIndex"`
    SdpMid        string `json:"sdpMid"`
}
```

### 3. 信令处理

**文件**: `internal/pkg/websocket/server.go`

```go
// handleAVCallMessage 处理音视频通话消息
func (s *Server) handleAVCallMessage(chatMessageReq *userreq.ChatMessageRequest) {
    var avData userreq.AVData
    if err := json.Unmarshal([]byte(chatMessageReq.AVdata), &avData); err != nil {
        zlog.Error("解析音视频通话数据失败: " + err.Error())
        return
    }

    message := &models.Message{
        Uuid:       fmt.Sprintf("M%s", util.GetNowAndLenRandomString(11)),
        SessionId:  chatMessageReq.SessionId,
        Type:       chatMessageReq.Type,
        SendId:     chatMessageReq.SendId,
        SendName:   chatMessageReq.SendName,
        SendAvatar: normalizePath(chatMessageReq.SendAvatar),
        ReceiveId:  chatMessageReq.ReceiveId,
        Status:     message_status_enum.Unsent,
        CreatedAt:  time.Now(),
        AVdata:     chatMessageReq.AVdata,
    }

    // 特定类型的音视频通话消息需要存储
    if avData.MessageId == "PROXY" && 
       (avData.Type == "start_call" || avData.Type == "receive_call" || avData.Type == "reject_call") {
        if err := s.messageDAO.CreateMessage(message); err != nil {
            zlog.Error("保存音视频通话消息失败: " + err.Error())
            return
        }
    }

    // 只处理单聊音视频通话
    if chatMessageReq.ReceiveId[0] == 'U' {
        messageRsp := userresp.AVMessageRespond{
            SessionId:  message.SessionId,
            SendId:     message.SendId,
            SendName:   message.SendName,
            SendAvatar: message.SendAvatar,
            ReceiveId:  message.ReceiveId,
            Type:       message.Type,
            AVdata:     message.AVdata,
            CreatedAt:  message.CreatedAt.Format("2006-01-02 15:04:05"),
        }

        jsonMessage, _ := json.Marshal(messageRsp)
        messageBack := &MessageBack{
            Message: jsonMessage,
            Uuid:    message.Uuid,
        }

        s.mutex.Lock()
        // 发送给接收者
        if receiveClient, ok := s.Clients[message.ReceiveId]; ok {
            receiveClient.SendBack <- messageBack
        }
        // 音视频通话不回显给发送者
        s.mutex.Unlock()
    }
}
```

### 4. 处理流程

```
1. 前端发送音视频消息 (type=3)
       ↓
2. WebSocket 接收消息
       ↓
3. 发送到 Kafka 消息队列
       ↓
4. Kafka Consumer 消费消息
       ↓
5. handleAVCallMessage 处理
       ↓
6. 存储消息（如果是关键信令）
       ↓
7. 转发给接收方客户端
```

#### 关键方法

**获取本地媒体流**
```javascript
const getLocalStream = async () => {
  try {
    if (localStream.value) {
      localStream.value.getTracks().forEach(track => track.stop())
    }
    const stream = await navigator.mediaDevices.getUserMedia({
      video: true,
      audio: true
    })
    localStream.value = stream
    return stream
  } catch (error) {
    // 处理不同类型的错误
    if (error.name === 'NotReadableError') {
      throw new Error('设备被占用')
    } else if (error.name === 'NotAllowedError') {
      throw new Error('权限被拒绝')
    }
    throw error
  }
}
```

**创建 RTCPeerConnection**
```javascript
const createPeerConnection = () => {
  peerConnection = new RTCPeerConnection(iceServers)

  // 添加本地轨道
  if (localStream.value) {
    localStream.value.getTracks().forEach(track => {
      peerConnection.addTrack(track, localStream.value)
    })
  }

  // 接收远程轨道
  peerConnection.ontrack = (event) => {
    if (event.streams && event.streams[0]) {
      remoteStream.value = event.streams[0]
    }
  }

  // ICE 候选处理
  peerConnection.onicecandidate = (event) => {
    if (event.candidate) {
      sendSignal('ice_candidate', {
        candidate: event.candidate.candidate,
        sdpMLineIndex: event.candidate.sdpMLineIndex,
        sdpMid: event.candidate.sdpMid
      })
    }
  }

  // 连接状态监听
  peerConnection.onconnectionstatechange = () => {
    if (peerConnection.connectionState === 'connected') {
      callStatus.value = 'connected'
      startCallTimer()
    } else if (peerConnection.connectionState === 'disconnected' ||
               peerConnection.connectionState === 'failed') {
      endCall()
    }
  }
}
```

**发起通话**
```javascript
const startCall = async (sessionId, receiveId, receiveInfo) => {
  sessionInfo.value = { sessionId, receiveId, receiveInfo }
  await getLocalStream()
  createPeerConnection()
  sendSignal('start_call')
  callStatus.value = 'calling'
  isIncoming.value = false
}
```

**接受通话**
```javascript
const acceptCall = async () => {
  await getLocalStream()
  createPeerConnection()
  sendSignal('accept_call')
  callStatus.value = 'calling'
}
```

**信令处理**
```javascript
const handleSignal = async (data) => {
  if (data.type !== 3) return

  const avData = JSON.parse(data.av_data)

  switch (avData.type) {
    case 'start_call':
      // 收到来电
      isIncoming.value = true
      callerInfo.value = { id: data.send_id, name: data.send_name, avatar: data.send_avatar }
      callStatus.value = 'ringing'
      break

    case 'accept_call':
      // 对方接听，创建 offer
      const offer = await peerConnection.createOffer()
      await peerConnection.setLocalDescription(offer)
      sendSignal('offer', { sdp: { type: offer.type, sdp: offer.sdp } })
      break

    case 'offer':
      // 收到 offer，创建 answer
      await peerConnection.setRemoteDescription(new RTCSessionDescription(avData.sdp))
      const answer = await peerConnection.createAnswer()
      await peerConnection.setLocalDescription(answer)
      sendSignal('answer', { sdp: { type: answer.type, sdp: answer.sdp } })
      break

    case 'answer':
      // 收到 answer
      await peerConnection.setRemoteDescription(new RTCSessionDescription(avData.sdp))
      break

    case 'ice_candidate':
      // 添加 ICE 候选
      await peerConnection.addIceCandidate(new RTCIceCandidate(avData.candidate))
      break

    case 'end_call':
    case 'reject_call':
      // 结束通话
      resetCall()
      break
  }
}
```

### 完整通话流程

```
发起方 (A)                              接收方 (B)
    │                                       │
    │  1. 点击通话按钮                       │
    ├─ 2. getUserMedia()                    │
    ├─ 3. 创建 RTCPeerConnection           │
    ├─ 4. 发送 start_call ────────────────→ │
    ├─ 5. 状态: calling                     │
    │                                       ├─ 6. 显示来电弹窗
    │                                       ├─ 7. 用户点击接听
    │                                       ├─ 8. getUserMedia()
    │                                       ├─ 9. 创建 RTCPeerConnection
    │                                       ├─ 10. 发送 accept_call ←─────────┤
    ├─ 11. 收到 accept_call                 │
    ├─ 12. createOffer()                    │
    ├─ 13. setLocalDescription(offer)       │
    ├─ 14. 发送 offer ────────────────────→ │
    │                                       ├─ 15. 收到 offer
    │                                       ├─ 16. setRemoteDescription(offer)
    │                                       ├─ 17. createAnswer()
    │                                       ├─ 18. setLocalDescription(answer)
    │                                       ├─ 19. 发送 answer ←──────────────┤
    ├─ 20. 收到 answer                      │
    ├─ 21. setRemoteDescription(answer)     │
    │                                       │
    ├─ 22. 收集 ICE 候选                    ├─ 22. 收集 ICE 候选
    ├─ 23. 发送 ice_candidate ────────────→ │
    │  ←──────────────────────────────── 发送 ice_candidate
    ├─ 24. 添加远程 ICE 候选                │
    │                                       │
    ├─ 25. P2P 连接建立成功                 │
    ├─ 26. 显示通话界面                     ├─ 26. 显示通话界面
    │                                       │
    ├─ 27. 音视频传输开始                   │
    │                                       │
    ├─ 28. 用户点击挂断                     │
    ├─ 29. 发送 end_call ─────────────────→ │
    ├─ 30. 关闭连接                         ├─ 31. 收到 end_call
    │                                       ├─ 32. 关闭连接
```

### 信令类型说明

| 信令类型 | 说明 | 数据 |
|---------|------|------|
| `start_call` | 发起通话邀请 | 无 |
| `accept_call` | 接受通话 | 无 |
| `reject_call` | 拒绝通话 | 无 |
| `offer` | SDP Offer | `{ sdp: { type, sdp } }` |
| `answer` | SDP Answer | `{ sdp: { type, sdp } }` |
| `ice_candidate` | ICE 候选 | `{ candidate, sdpMLineIndex, sdpMid }` |
| `end_call` | 挂断通话 | 无 |

## 常见问题

### 1. 设备被占用 (NotReadableError)

**原因**: 同一设备的摄像头不能同时被多个应用/标签页使用

**解决方案**:
- 使用两台不同设备测试
- 关闭其他使用摄像头的应用（Zoom、Teams等）
- 使用不同的浏览器

### 2. 权限被拒绝 (NotAllowedError)

**原因**: 用户拒绝了摄像头/麦克风权限

**解决方案**:
- 在浏览器设置中允许权限
- 使用 HTTPS（getUserMedia 要求安全上下文）

### 3. P2P 连接失败

**原因**: NAT 穿透失败，通常需要 TURN 服务器

**解决方案**:
- 添加 TURN 服务器配置
- 使用付费 TURN 服务（如 Twilio、Xirsys）

```javascript
const iceServers = {
  iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    {
      urls: 'turn:your-turn-server.com:3478',
      username: 'username',
      credential: 'password'
    }
  ]
}
```

### 4. 没有视频画面

**检查项**:
- 确认摄像头权限已授予
- 检查 `localStream.value` 是否有值
- 确认 `<video>` 元素的 `srcObject` 已设置
- 检查是否有其他应用占用摄像头

### 5. 只有单向视频

**原因**: 通常是一方的 `addTrack` 或 `ontrack` 处理有问题

**检查项**:
- 确认双方都调用了 `addTrack`
- 确认双方都监听了 `ontrack` 事件
- 检查 ICE 候选是否正确交换
