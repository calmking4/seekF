# SeekF 聊天功能实现详解

## 一、架构概览

SeekF 的聊天功能采用 **WebSocket + Kafka + MySQL + Redis** 的架构设计，支持以下特性：

- **实时消息推送**：基于 WebSocket 长连接
- **消息队列**：使用 Kafka 解耦消息发送与处理
- **消息持久化**：MySQL 存储聊天记录
- **缓存加速**：Redis 缓存最近消息列表
- **消息类型**：支持文本、文件、音视频通话
- **聊天模式**：支持单聊和群聊

## 三、核心组件详解

### 3.1 WebSocket 服务器 (Server)

**文件位置**: `internal/pkg/websocket/server.go`

```go
type Server struct {
    Clients map[string]*Client  // 在线客户端映射表
    mutex   *sync.Mutex         // 并发安全锁
    Login   chan *Client        // 登录通道
    Logout  chan *Client        // 退出登录通道
}
```

**工作流程**:

1. **启动服务器**: `Start()` 方法启动两个协程
   - 主协程：监听 Login/Logout 通道处理连接
   - Kafka 消费协程：`readKafkaMessages()` 持续读取消息

2. **客户端连接管理**:
   - `handleLogin()`: 将客户端加入 Clients 映射表
   - `handleLogout()`: 从 Clients 映射表移除客户端

### 3.2 WebSocket 客户端 (Client)

**文件位置**: `internal/pkg/websocket/client.go`

```go
type Client struct {
    Conn     *websocket.Conn   // WebSocket 连接
    Uuid     string            // 用户唯一标识
    SendBack chan *MessageBack // 接收服务器的消息通道
}
```

**双协程模型**:

- **Read() 协程**: 持续读取客户端消息 → 发送到 Kafka
- **Write() 协程**: 从 SendBack 通道读取 → 推送给客户端

```go
// Read 流程
for {
    _, jsonMessage, err := c.Conn.ReadMessage()  // 阻塞读取
    // ... 解析消息
    SendMessageToKafka(jsonMessage)  // 发送到 Kafka
}

// Write 流程
for messageBack := range c.SendBack {  // 阻塞等待
    c.Conn.WriteMessage(websocket.TextMessage, messageBack.Message)
    UpdateMessageStatus(messageBack.Uuid)  // 更新为已发送
}
```

### 3.3 Kafka 消息队列

k.ChatWriter = &kafka.Writer{

**作用**:

## 五、单聊与群聊实现

### 5.1 单聊 (sendToUser)

```go
func (s *Server) sendToUser(message *models.Message, messageBack *MessageBack, ...) {
    s.mutex.Lock()
    // 推送给接收者
    if receiveClient, ok := s.Clients[message.ReceiveId]; ok {
        receiveClient.SendBack <- messageBack
    }
    // 回显给发送者
    if sendClient, ok := s.Clients[message.SendId]; ok {
        sendClient.SendBack <- messageBack
    }
    s.mutex.Unlock()
    
    // 更新Redis缓存
    s.updateUserMessageListCache(...)
}
```

**特点**:
- 接收者在线则实时推送
- 发送者收到回显确认
- 更新双方的消息列表缓存

### 5.2 群聊 (sendToGroup)

```go
func (s *Server) sendToGroup(message *models.Message, messageBack *MessageBack, ...) {
    // 1. 查询群组成员
    var group models.GroupInfo
    db.GormDB.Where("uuid = ?", message.ReceiveId).First(&group)
    
    // 2. 解析成员列表
    var members []string
    json.Unmarshal(group.Members, &members)
    
    // 3. 推送给所有在线成员
    s.mutex.Lock()
    for _, member := range members {
        if client, ok := s.Clients[member]; ok {
            client.SendBack <- messageBack
        }
    }
    s.mutex.Unlock()
    
    // 4. 更新群组缓存
    s.updateGroupMessageListCache(...)
}
```

**特点**:
- 从数据库查询群组成员
- 只推送给在线成员
- 离线成员通过历史记录拉取

## 六、Redis 缓存策略

### 6.1 缓存设计

| 缓存 Key | 说明 | 过期时间 |
|---------|------|---------|
| `message_list_{sendId}_{receiveId}` | 单聊消息列表 | 30分钟 |
| `group_messagelist_{groupId}` | 群聊消息列表 | 30分钟 |

### 6.2 缓存更新

```go
func (s *Server) updateUserMessageListCache(sendId, receiveId string, messageRsp ...) {
    // 1. 尝试获取缓存
    rspString, err := myredis.GetKeyNilIsErr("message_list_" + sendId + "_" + receiveId)
    
    if err == nil {
        // 2. 解析现有缓存
        var rsp []userresp.GetMessageListRespond
        json.Unmarshal([]byte(rspString), &rsp)
        
        // 3. 追加新消息
        rsp = append(rsp, *messageRsp)
        
        // 4. 重新设置缓存
        rspByte, _ := json.Marshal(rsp)
        myredis.SetKeyEx("message_list_"+sendId+"_"+receiveId, string(rspByte), time.Minute*30)
    }
}
```

**策略**:
- 缓存最近的消息列表
- 新消息到达时追加更新
- 缓存过期后从数据库重新加载

## 十、性能优化点

1. **连接池**: WebSocket 连接使用通道缓冲避免阻塞
2. **消息队列**: Kafka 解耦处理逻辑，支持水平扩展
3. **缓存策略**: Redis 缓存热点数据，减少数据库查询
4. **并发控制**: 使用 `sync.Mutex` 保护 Clients 映射表
5. **协程模型**: 每个客户端两个协程，读写分离
