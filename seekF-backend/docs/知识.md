# 1.Bind ShouldBind ShouldBindJson的区别

| 特性             | Bind                          | ShouldBind                 | ShouldBindJSON                 |
| ---------------- | ----------------------------- | -------------------------- | ------------------------------ |
| **格式适配**     | 自动识别（JSON/Form/XML 等）  | 自动识别（同 Bind）        | 仅支持 JSON 格式               |
| **失败处理**     | 自动返回 400 Bad Request 响应 | 仅返回错误，不主动返回响应 | 仅返回错误，不主动返回响应     |
| **开发者控制权** | 低（框架接管响应）            | 高（自行处理错误 / 响应）  | 高（自行处理错误 / 响应）      |
| **底层依赖**     | 基于 ShouldBind 封装          | 核心绑定逻辑               | 基于 ShouldBind 限定 JSON 格式 |
| **适用场景**     | 简单场景，接受默认 400 响应   | 复杂场景，需自定义错误逻辑 | 仅 JSON 请求，需自定义错误     |

`Bind` 底层复用 `ShouldBind` 的逻辑，其对结构体标签的要求和 `ShouldBind` 完全一致：

|             支持的请求格式              | 结构体需要的标签 |         标签示例          |
| :-------------------------------------: | :--------------: | :-----------------------: |
|        JSON（application/json）         | `json:"字段名"`  | `Uuid string json:"uuid"` |
| 表单（form-data/x-www-form-urlencoded） | `form:"字段名"`  | `Uuid string form:"uuid"` |
|         XML（application/xml）          |  `xml:"字段名"`  | `Uuid string xml:"uuid"`  |
