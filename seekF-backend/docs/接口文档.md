# seekF-backend 接口文档

本文档根据当前代码（`internal/router/router.go`、`internal/controllers/user/*`、`internal/dto/user/*`、`internal/services/user_service/*`）整理。

## 基础信息

- **服务端口**：代码 `main.go` 使用 `r.Run()`，Gin 默认监听 `:8080`（若你在其他地方通过环境变量/参数覆盖端口，请以实际运行配置为准）
- **统一前缀**：`/user`
- **请求方式**：目前接口均为 `POST`
- **Content-Type**：`application/json`

## 鉴权说明（JWT）

除 `register/login` 外，其余接口都需要在请求头带上：

- **Header**：`Authorization: Bearer <token>`
- **token 来源**：登录接口 `/user/login` 的响应字段 `data.token`

## 通用响应结构

后端统一返回 HTTP 200，但通过 JSON 字段 `code` 表示业务状态：

- **成功**：

```json
{
  "code": 200,
  "message": "xxx",
  "data": {}
}
```

- **失败**（通常 `code` 为 400/401/403/500 等，且一般没有 `data`）：

```json
{
  "code": 400,
  "message": "错误原因"
}
```

## 1. 用户注册

- **方法**：POST
- **Path**：`/user/register`
- **鉴权**：否

### Request Body（JSON）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `nickname` | string | 是 | 昵称 |
| `telephone` | string | 是 | 手机号 |
| `password` | string | 是 | 密码 |

示例：

```json
{
  "nickname": "张三",
  "telephone": "13800000000",
  "password": "123456"
}
```

### Response

成功示例：

```json
{
  "code": 200,
  "message": "注册成功"
}
```

## 5. 更新用户信息

- **方法**：POST
- **Path**：`/user/userinfo/updateUserInfo`
- **鉴权**：是
- **权限限制**：只能更新自己的信息（`req.uuid` 必须等于 token 内的 `Uuid`）

### Request Body（JSON）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `uuid` | string | 否 | 用户 UUID（当前实现未强制 required，但会用于“只能更新自己”校验） |
| `email` | string | 否 | 邮箱 |
| `nickname` | string | 否 | 昵称 |
| `birthday` | string | 否 | 生日（字符串） |
| `signature` | string | 否 | 个性签名 |
| `avatar` | string | 否 | 头像 URL |

示例：

```json
{
  "uuid": "Uxxxxxxxxxxx",
  "nickname": "新昵称",
  "avatar": "https://example.com/a.png"
}
```

### Response

成功示例：

```json
{
  "code": 200,
  "message": "更新用户信息成功"
}
```

## 6. 创建群聊

- **方法**：POST
- **Path**：`/user/group/createGroup`
- **鉴权**：是
- **说明**：`owner_id` 会在后端被强制设置为当前登录用户 UUID

### Request Body（JSON）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `owner_id` | string | 否 | 群主 UUID（会被覆盖） |
| `name` | string | 否 | 群名 |
| `notice` | string | 否 | 群公告 |
| `add_mode` | int | 否 | 加群方式（当前业务里 0 常用于“直接加入”） |
| `avatar` | string | 否 | 群头像 |

示例：

```json
{
  "name": "测试群",
  "notice": "欢迎加入",
  "add_mode": 0,
  "avatar": ""
}
```

### Response

```json
{
  "code": 200,
  "message": "创建群聊成功"
}
```

## 12. 移除群聊成员

- **方法**：POST
- **Path**：`/user/group/removeGroupMembers`
- **鉴权**：是
- **权限限制**：只有群主可移除；且不能移除群主

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `group_id` | string | 否 | 群 UUID |
| `uuid_list` | string[] | 否 | 要移除的用户 UUID 列表 |

示例：

```json
{
  "group_id": "Gxxxxxxxxxxx",
  "uuid_list": ["Uaaaa", "Ubbbb"]
}
```

### Response

```json
{

  "message": "移除群聊成员成功"
```
