# 断点续传功能实现分析

## 1. 功能概述

断点续传功能主要用于大文件（视频）上传场景，允许用户在网络中断或主动暂停后，从已上传的位置继续上传，而不需要重新上传整个文件。

#### 查询已上传分片（断点续传核心）
```go
// MultipartListParts 查询已上传的分片列表（用于断点续传）
func MultipartListParts(ctx context.Context, objectKey, uploadId string) ([]PartInfo, error) {
    client, err := getClient()
    if err != nil {
        return nil, err
    }

    var allParts []PartInfo
    partNumberMarker := int32(0)

    for {
        result, err := client.ListParts(ctx, &oss.ListPartsRequest{
            Bucket:           oss.Ptr(ossBucketName),
            Key:              oss.Ptr(objectKey),
            UploadId:         oss.Ptr(uploadId),
            MaxParts:         1000,
            PartNumberMarker: partNumberMarker,
        })
        if err != nil {
            return nil, fmt.Errorf("查询已上传分片失败: %w", err)
        }

        for _, p := range result.Parts {
            etag := ""
            if p.ETag != nil {
                etag = strings.Trim(*p.ETag, `"`)
            }
            allParts = append(allParts, PartInfo{
                PartNumber: p.PartNumber,
                ETag:       etag,
                Size:       p.Size,
            })
        }

        if !result.IsTruncated {
            break
        }
        partNumberMarker = result.NextPartNumberMarker
    }

    sort.Slice(allParts, func(i, j int) bool {
        return allParts[i].PartNumber < allParts[j].PartNumber
    })

    return allParts, nil
}
```

 * 生成文件指纹，用于断点续传识别同一文件

function getFileFingerprint(file) {
    return `${file.name}_${file.size}_${file.lastModified}`

#### 本地状态存储
```javascript
const STORAGE_PREFIX = 'seekf_video_upload_'

function loadUploadState(fingerprint) {
    const raw = localStorage.getItem(getStorageKey(fingerprint))
    return raw ? JSON.parse(raw) : null
}

function saveUploadState(fingerprint, state) {
    localStorage.setItem(getStorageKey(fingerprint), JSON.stringify(state))
}
```

### 4.3 上传流程

```javascript
export async function uploadVideoResumable(file, fileType, callbacks = {}) {
    const fingerprint = getFileFingerprint(file)
    let state = loadUploadState(fingerprint)

    // 1. 尝试恢复已有上传任务
    if (state?.uploadId && state?.objectKey) {
        // 调用 /upload/status 查询已上传分片
        const statusRes = await $fetch(`${apiBase}/user/file/upload/status`, {
            body: { upload_id: state.uploadId, object_key: state.objectKey }
        })
        // 更新本地状态
        state.parts = statusRes.data.parts
        saveUploadState(fingerprint, state)
    }

    // 2. 初始化新任务（如果没有）
    if (!state?.uploadId) {
        const initRes = await $fetch(`${apiBase}/user/file/upload/init`, {
            body: { file_name: file.name, file_type: fileType, ... }
        })
        state = { uploadId: initRes.data.uploadId, ... }
        saveUploadState(fingerprint, state)
    }

    // 3. 上传缺失分片
    const uploadedPartNumbers = new Set(state.parts.map(p => p.partNumber))
    for (let partNumber = 1; partNumber <= totalParts; partNumber++) {
        if (uploadedPartNumbers.has(partNumber)) continue  // 跳过已上传

        // 切片并上传
        const chunk = file.slice(start, end)
        await $fetch(`${apiBase}/user/file/upload/chunk`, { body: formData })

        // 更新状态
        state.parts.push({ partNumber, etag, size })
        saveUploadState(fingerprint, state)
    }

    // 4. 合并分片
    await $fetch(`${apiBase}/user/file/upload/complete`, {
        body: { upload_id, object_key, parts }
    })

    // 5. 清理本地状态
    clearUploadState(fingerprint)
}
```

## 6. 关键设计点

### 6.1 文件指纹
- 使用 `文件名 + 文件大小 + 最后修改时间` 生成唯一标识
- 同一文件在不同时间选择会生成不同指纹

### 6.2 状态持久化
- 使用 localStorage 存储上传状态
- 页面刷新后可恢复上传进度

### 6.3 分片大小
- 默认 5MB，由后端初始化时返回
- 前端根据返回的 partSize 进行切片

### 6.4 错误处理
- `UploadPausedError`: 暂停错误，不清理状态
- `AbortError`: 取消错误，清理状态
- 其他错误: 显示重试按钮

### 6.5 并发控制
- 分片串行上传，避免并发请求过多
- 可通过 Promise.allSettled 实现有限并发

## 9. 注意事项

1. **分片最小大小**: OSS 要求除最后一个分片外，其他分片最小 100KB
2. **最大分片数**: OSS 限制最多 10000 个分片
3. **ETag 处理**: OSS 返回的 ETag 带引号，需要去除
4. **超时处理**: 大文件上传需要设置较长的超时时间
5. **并发上传**: 当前实现为串行上传，可根据需要改为并发
